# 《岁己：空响撤离》游戏设计草案

> 英文工作名：**SUI: Echoes Below**  
> 核心幻想：像一只敏捷的小鸟一样深入陌生空洞，在风险越来越高的远征中搜集装备与世界真相，并把故事安全带回“饼干岁”们的直播间。

## 一句话玩法

类《空洞骑士》的二维房间式动作探索手感，加上“带装备进图、搜刮、战斗、权衡风险、从撤离点安全回家”的远征循环。

## 世界与主角

岁己 SUI 原本只是在给饼干岁们直播，追逐一段不属于任何频道的神秘鸟鸣时，连人带直播设备坠入了“空响世界”。这里的废墟会重复记忆，死去的探路者会留下回声，而远方有一座仍在向现实发送信号的古老天线。

岁己要把散落在各区域的“回声羽片”带回饼干台，修好直播信标，找到回去的路。她仍会嘴硬地把每次危险远征说成“普通户外直播”，基地里看不见的饼干岁则用弹幕、补给箱和路标回应她。

视觉不直接复刻任何现有作品：采用深蓝、青绿、暖金的洞窟剪影，角色与道具用本地岁己素材、Emoji 和几何图形混合表现，通过 squash & stretch、残影、粒子和镜头反馈建立自己的辨识度。

## 设计支柱

1. **动作要轻快**：跑、跳、下劈、冲刺和击退都必须即时、清晰、可预测。
2. **每次深入都是选择**：背包越满、装备越好，继续深入的收益与死亡风险就越高。
3. **失败会痛但不劝退**：当局装备掉落；下一轮可找回一次；基地永久进度不会倒退。
4. **目标始终可读**：未知地图仍显示区域轮廓、自己的相对位置和目标方向；拿到导航羽片后才显示房间细节、捷径与撤离点。
5. **复杂度来自组合，不来自表格**：武器、护甲、头部、鞋各自只有少量清楚的属性或特效。

## 核心循环

```text
饼干台整备
  → 选择武器 / 护甲 / 头部 / 鞋 / 背包，并整理随身物品
  → 从已解锁入口进入地图
  → 平台探索 + 战斗 + 开箱 + 收集
  → 背包与风险抉择
      ↘ 前往撤离点：携包返回基地，再卸入安全仓库，推进永久目标
      ↘ 死亡：当局物品留在死亡地点，下一局仅可找回一次
  → 修甲 / 制造 / 扩仓 / 解锁捷径
  → 挑战更深区域
```

## 装备与物品

### 装备槽

| 槽位 | 作用 | Demo 示例 |
| --- | --- | --- |
| 武器 | 攻击力、范围、攻速与攻击特效 | 旧羽钉：均衡；回声长针：长范围；雷羽短刃：短范围、连击快 |
| 护甲 | 提供可修复的蓝色护甲值 | 旧直播甲：+2 蓝甲；矿工壳：+4 蓝甲但略减速 |
| 头部 | 一个易懂的被动效果 | 小猫帽：受伤后短暂加速；测绘镜：更早显示地图细节 |
| 鞋 | 移动能力或探索效果 | 软羽靴：移速增加；影步靴：解锁穿敌黑冲 |
| 背包 | 决定远征中的二维携行空间 | 折羽背包：4×5；后续测绘员背架：5×6 |
| 收集品 | 不占装备槽，解锁故事/结局/制造 | 导航羽片、回声核心、饼干岁留言 |

护甲优先承受伤害，可在基地花“空响尘”修复。生命归零则远征失败。

### 背包

- 随身背包是装备，而不是基地仓库的别名；Demo 初始“折羽背包”为 4×5。
- 物品按矩形占格：空响尘 1×1、修补片 1×2、普通装备约 2×2～3×3、回声核心 3×3；材料可以有限堆叠。
- 基地仓库初始为 9×10，可永久扩建为 10×10；留在这里的物品不参与远征死亡结算。
- 地上物品可查看名称、稀有度和用途，再决定是否拾取。
- 背包缺少连续空格时会明确显示所需尺寸；撤离后物品仍留在背包，玩家在基地拖入仓库。

## 难度曲线与地图

第一张地图“寂羽空洞”是一张 3,200×2,200 的二维房间网络，由横向房间、纵向竖井和折返支路组成：

| 区域 | 定位 | 敌人 / 收益 | 引导 |
| --- | --- | --- | --- |
| 失落前庭 | 出生点附近，教学与安全撤离 | 缓慢爬虫、基础材料 | 明亮灯笼、近距离撤离点 |
| 荧菌裂谷 | 中段，首次明确风险升级 | 飞蛾、远程孢子，绿色装备 | 未解锁地图时显示轮廓与目标光柱 |
| 静默机房 | 深处高风险区 | 精英守卫、紫色装备、回声核心 | 解锁捷径后可从前庭电梯快速抵达 |

老手便利：解锁“旧维护电梯”后可选择深层入口；未来地图也沿用入口解锁接口。

## 目标与首个结局

Demo 的目标链始终显示在基地和地图页：

1. 完成第一次撤离，把任意战利品带回饼干台。
2. 带回导航羽片，解锁寂羽空洞的完整地图。
3. 带回足够空响尘，升级一次仓库。
4. 从荧菌裂谷开启维护电梯捷径。
5. 击败静默机房的“失频守卫”，带回回声核心。
6. 在基地启动直播信标，播放首个结局动画“收到请回答”。

结局保留分支字段，未来可以根据是否收集全部留言、是否修复某件设备、是否击败隐藏敌人进入不同结局。

## 存档与联网边界

- `SaveRepository` 负责浏览器 `localStorage` 自动存档、版本迁移、导出和导入。
- 游戏逻辑通过领域事件描述关键行为，例如 `raid.started`、`loot.collected`、`player.died`、`raid.extracted`。
- 当前使用 `LocalProfileService`；未来替换为 `RemoteProfileService` 即可同步账号、仓库和远征结算。
- 战斗状态、地图定义和掉落表使用稳定 ID，不把 Phaser 对象直接写入存档或网络包。
- 多地图由 `MapDefinition` 注册表加载；Demo 只注册 `hollow_01`。

## 技术选型（2026-07）

- **Vite 8 + React 19 + TypeScript**：React 负责基地、背包、菜单与存档 UI。
- **Phaser 4.2.1**：负责 2D 世界、Arcade Physics、输入、相机、粒子与场景生命周期。
- Phaser 是完整 2D 网页游戏框架，并支持 React；PixiJS 8 更偏渲染层，若采用它还需要自行补齐大量游戏系统。参考：[Phaser 介绍](https://docs.phaser.io/phaser/getting-started/what-is-phaser)、[Phaser 4 发布列表](https://phaser.io/download/phaser4)、[PixiJS 架构](https://pixijs.com/8.x/guides/concepts/architecture)。
